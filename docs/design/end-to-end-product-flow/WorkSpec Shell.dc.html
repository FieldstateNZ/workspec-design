What changed — and why
01 · NAV
Grouped by what a surface is
A flat list of six views became four roles: Board · Design · Registers · Deliver. You read the app's shape from the rail.
02 · BOARD
Discovery is one board, two lenses
Freeform and Structured aren't two tabs — they're a lens toggle on the same canvas. Sketch and shape never leave home.
03 · DESIGN
Design rooms are tethered, not scattered
Architecture, Data Dictionary, Journeys, Data Flow group together — entered from an element, with a way back. Topology links in from the Workspace section at the foot of the nav.
04 · NO PHASE
The phase tag is gone
No "Phase: Discovery." Readiness is per element. Atlas proposes structure live; you accept to ink.
05 · ATLAS
One co-author rail, always reachable
"Proposals" becomes Atlas: a reasoning stream + pencilled drafts on canvas. Quiet when idle, loud while shaping.

R1 · THE CONTRADICTION
Design isn't one kind of surface
The Arc's nuance now lives in the nav: Data Dictionary + User Journeys = ON BOARD — lenses that re-project the same board, clustering logic shared. Architecture + Data Flow = ROOMS ↗ — wire-routed, own zoom, tethered. One group ("Design"), two behaviours, labelled per item.
R2 · "BEING REWORKED"
An overlay, not a fourth state
drafted / inked / superseded stay untouched. The rework halo is board metadata: the element remains inked, links stay live but carry a contested flag — traceability and gap detection report "contested," never "dangling." No schema state added.
R3 · THE STACKING RULE
Four channels, four slots — never compete
Form = native lens · border = VC state · corner tag = arc stage · halo = transient. Worst case, all four at once:

R4 · MEMORY + DATA MODEL
Tombstones, and loosen ships first
Dismissals are tombstoned — remembered across sweeps; Atlas re-proposes only when its inputs materially change (the Coffers pattern). And the morph ships with placement = {x, y, lensOffset?} — the per-lens offset is nullable from day one, so "loosen" is a write, not a retrofit.
THE FOUR MECHANICS · PINNED Pill = lens. Total re-representation — every item exists in both, nothing vanishes. · Overlay = tint. Temporary, total — recolours, never filters. · ON BOARD = projection. Partial re-cluster, board logic. · ROOM ↗ = tethered surface. Own spatial logic, breadcrumb home. A partial view never goes in the pill — "lens" must never come to mean "a view that hides things."
